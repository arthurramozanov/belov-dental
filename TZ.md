# ТЗ: Лендинг «Стоматология им. Белова»

## 1. О проекте

**Что:** одностраничный сайт-визитка авторской премиум-стоматологии в Москве.

**Позиционирование:** не сетевая клиника, а частная практика доктора Белова — спокойная, неагрессивная, основанная на репутации и личном внимании. Главное конкурентное отличие — врач, а не оборудование.

**Целевая аудитория:** мужчины и женщины 30–55 лет, средний+/высокий доход, ценят время и сервис, выбирают по рекомендации, готовы платить выше рынка за предсказуемое качество и отсутствие конвейера.

**Ключевое сообщение:** «Стоматология, к которой возвращаются. И приводят детей».

**Цель лендинга:** запись на консультацию через форму или звонок. Вторичная цель — снять страх, дать почувствовать атмосферу клиники до визита.

## 2. Дизайн-система

### Цветовая палитра

| Назначение | HEX | Tailwind alias |
|---|---|---|
| Фон основной (тёплый off-white) | `#FAF8F4` | `bg-cream` |
| Фон карточек / контраст | `#FFFFFF` | `bg-white` |
| Текст основной (графит) | `#1A1A1A` | `text-ink` |
| Текст вторичный | `#5C5C5C` | `text-ink-muted` |
| Бордеры, разделители | `#E8E2D6` | `border-sand` |
| Акцент основной (приглушённый зелёный) | `#3A5A3F` | `bg-forest / text-forest` |
| Акцент вторичный (приглушённое золото) | `#C9A961` | `text-gold / bg-gold` |
| Акцент hover | `#2C4530` | `hover:bg-forest-dark` |

Палитра должна быть прописана как кастомные цвета в `tailwind.config.ts`. Основной акцент `forest` используется для CTA-кнопок и активных состояний. Золото — точечно: подчёркивание под цифрами достижений, иконка награды, разделитель в hero.

### Типографика

- **Заголовки:** `Playfair Display` (Google Fonts, variable, weight 400–700, italic поддержан). Подключить через `next/font/google` с subsets `['latin', 'cyrillic']`. Контрастный редакторский serif с полноценной кириллицей.
- **Body / UI:** `Inter` (Google Fonts, variable, weight 400–600). Подключить через `next/font/google` с subsets `['latin', 'cyrillic']`.
- В `tailwind.config.ts` зарегистрировать как `font-display` (Playfair Display) и `font-sans` (Inter, default).

**Шкала размеров (desktop):**
- H1 — 72px / line-height 1.05 / weight 400 / Playfair Display
- H2 — 48px / 1.1 / 400 / Playfair Display
- H3 — 28px / 1.2 / 500 / Playfair Display
- Body L — 19px / 1.6 / 400 / Inter
- Body — 17px / 1.6 / 400 / Inter
- Small / caption — 14px / 1.5 / 500 / Inter / letter-spacing 0.02em / uppercase для надзаголовков

**Важно про Playfair Display:** это контрастный шрифт (тонкие штрихи vs толстые). На мелких размерах <20px он становится плохо читаем. Не использовать его для body или caption — только для H1, H2, H3 и крупных декоративных акцентов (имена врачей, цены крупным кеглем, цитаты в отзывах).

**Mobile:** H1 → 44px, H2 → 32px, остальное масштабируется через `clamp()` или Tailwind breakpoints.

### Геометрия

- Скругления: `rounded-2xl` (16px) для карточек, `rounded-full` для кнопок и pill-элементов, `rounded-3xl` (24px) для крупных media-блоков.
- Бордеры: `1px solid #E8E2D6` для разделения.
- Тени: только `shadow-sm` с очень мягким blur, либо вообще без теней — премиум держится на бордерах и контрасте, не на drop-shadow.

### Отступы и сетка

- Контейнер: `max-w-[1280px]` с `px-6 md:px-8 lg:px-12`.
- Вертикальный ритм между секциями: `py-24 md:py-32 lg:py-40`.
- Внутри секций: `gap-8` для сеток, `gap-16` между ключевыми блоками.
- Сетка: 12 колонок, gap 32px на desktop, 24px на tablet, 16px на mobile.

### Кнопки

- **Primary:** фон `bg-forest`, текст `text-white`, `rounded-full`, `px-8 py-4`, hover — `bg-forest-dark` + лёгкий `scale-[1.02]`. Шрифт Inter 16px / weight 500 / letter-spacing 0.01em.
- **Secondary:** прозрачный фон, бордер `border border-ink`, текст `text-ink`, hover — фон `bg-ink text-white`.
- **Ghost (для ссылок):** только текст с `underline-offset-4` и подчёркиванием на hover.

### Анимации

Библиотека: **Framer Motion**.

- Появление блоков на scroll: `initial={{ opacity: 0, y: 24 }}`, `whileInView={{ opacity: 1, y: 0 }}`, `viewport={{ once: true, margin: "-100px" }}`, `transition={{ duration: 0.6, ease: [0.16, 1, 0.3, 1] }}`.
- Hover карточек: `transition: 200ms ease-out`, `translate-y-[-2px]`, замена бордера на `border-forest`.
- Hero — лёгкий parallax на изображении при скролле (через `useScroll` + `useTransform`).
- Карусель отзывов — fade-переход 400ms между слайдами, не slide.
- **Никаких** морфингов, бликов, мигающих border-glow, появлений по буквам — это не премиум.

## 8. Особенности (на чём не экономить)

1. **Hero:** parallax на портретное фото — обязательно. Это первое впечатление, оно делает всё.
2. **Типографика Playfair Display:** italic для второй строки H1 — обязательно. Это единственный «характер» в дизайне, без него получится плоско.
3. **Маска телефона** в форме — `react-input-mask` или ручная реализация. Формат `+7 (___) ___-__-__`.
4. **Валидация формы Zod** должна показывать ошибки под полями плавно (Framer Motion `AnimatePresence`).
5. **Бордеры вместо теней.** Премиум держится на бордерах `#E8E2D6` и контрасте, не на drop-shadow. Если возникнет соблазн добавить `shadow-lg` — не добавлять.
6. **Документальная фотография.** Если на Unsplash попадаются стоковые «улыбки на белом фоне» — пропускать. Берём только постановочные с естественным светом и интерьером.
7. **Lighthouse:** Performance ≥ 90, Accessibility ≥ 95. Картинки через `next/image` с `priority` на hero, остальные — `loading="lazy"`.
8. **Карусель отзывов:** fade, не slide. Slide — массовый сегмент, fade — премиум.
9. **Отзывы с реальной структурой** (имя + тип лечения + год прихода) — это сильно продаёт, не пропускать атрибуцию.

## 9. Что НЕ нужно делать

- Не подключать настоящий backend, базу данных, API для отправки форм — только Toast-имитация.
- Не делать авторизацию, личный кабинет, админку.
- Не подключать платежи, онлайн-запись с реальным календарём (только форма).
- Не использовать Material UI, Ant Design, Chakra — только shadcn.
- Не делать тёмную тему (light only).
- Не использовать gradient backgrounds, glassmorphism, neumorphism.
- Не использовать стоковые иконки услуг типа «3D-зубики» — только Lucide.
- Не добавлять чат-виджет, всплывающие окна «акция -50%», cookie-баннер с кнопкой во весь экран. Cookie-уведомление — минимальная полоса снизу с одной кнопкой «Принять» (опционально, можно вообще опустить).
- Не делать слайдер на hero. Один статичный hero с одним заголовком.
